Présentation 概要

▶ Problématique

Comment centraliser la gestion des utilisateurs et des droits d'accès sur un réseau d'entreprise, tout en appliquant des stratégies de sécurité uniformes sur tous les postes clients ?

◈ Contexte du projet

Projet d'administration système en 1ère année de BTS SIO option SISR. L'objectif est de simuler un environnement d'entreprise réaliste avec un contrôleur de domaine, des comptes utilisateurs, des groupes de sécurité et des stratégies de groupe (GPO), hébergé sur VMware Workstation.

Windows Server 2022 VMware Workstation Active Directory GPO

// 02

Objectifs techniques 目標

  • ▸ Installer et configurer Windows Server 2022 en machine virtuelle SRV-AD-01
  • ▸ Déployer le rôle AD DS et créer le domaine sisr.local
  • ▸ Créer une structure organisationnelle (OU Utilisateurs-SISR, comptes, groupes)
  • ▸ Appliquer une stratégie de groupe GPO-Employes pour restreindre les accès
  • ▸ Joindre un poste client Windows 10 CLI-WIN-01 au domaine
  • ▸ Valider la jonction et l'application des GPO sur le client via gpresult /r

Réalisation technique 実装

ÉTAPE 01

Préparation des machines virtuelles

Deux étapes sont indispensables avant toute installation de rôle : attribuer un nom explicite au serveur (convention : SRV = serveur, AD = Active Directory, 01 = premier de la série) et lui assigner une adresse IP fixe — obligatoire pour un contrôleur de domaine, car les postes clients doivent toujours pouvoir le joindre à la même adresse.

SRV-AD-01 est créé avec deux cartes réseau : une carte NAT pour l'accès Internet et une carte VMnet2 (Host-only) avec l'IP fixe 192.168.2.10/24. Le champ DNS préféré est laissé vide à ce stade — il sera renseigné après l'installation du rôle DNS.

Ce point du DNS sera à l'origine d'une difficulté rencontrée lors de la jonction du poste client (voir section 06).
ÉTAPE 02

Installation du rôle AD DS

Le rôle AD DS (Active Directory Domain Services) est le service central qui transforme le serveur en contrôleur de domaine. L'installation se déroule en deux temps :

1. Ajout du rôle via Gestionnaire de serveur → Gérer → Ajouter des rôles et fonctionnalités → sélection de « Services de domaine Active Directory ». Les dépendances (outils d'administration AD, module PowerShell AD) sont installées automatiquement.

2. Promotion en contrôleur de domaine via le lien de notification post-installation. On sélectionne « Ajouter une nouvelle forêt » avec le nom de domaine racine sisr.local.

Le suffixe .local indique un domaine privé non résolvable sur Internet. Un mot de passe DSRM est défini pour l'accès en mode récupération. Le rôle DNS est installé automatiquement — indispensable pour que les clients localisent le DC via des enregistrements SRV.
ÉTAPE 03

Création de la structure organisationnelle

Une Unité d'Organisation (OU) est un conteneur logique permettant de regrouper des objets selon des critères métier et d'y lier des GPO ciblées. Via la console Utilisateurs et ordinateurs Active Directory, on crée :

— L'OU Utilisateurs-SISR directement sous sisr.local
— Les comptes jean.dupont@sisr.local et Marie.Martine@sisr.local
— Le groupe de sécurité GRP-Employes (type Sécurité, étendue Domaine local)

Les deux utilisateurs sont ajoutés comme membres du groupe, ce qui permettra de filtrer l'application des GPO.

ÉTAPE 04

Configuration de la GPO

Via la console GPMC, on crée l'objet GPO-Employes et on le lie à l'OU Utilisateurs-SISR. Le paramètre activé est :

// Chemin dans l'éditeur GPO :
Configuration utilisateur
  → Modèles d'administration
    → Panneau de configuration
      → Interdire l'accès au Panneau de configuration et à l'application Paramètres du PC
 
Statut : Activé ✓
S'applique à la prochaine ouverture de session — ou immédiatement après gpupdate /force. Le filtre de sécurité cible par défaut les « Utilisateurs authentifiés » de l'OU.
ÉTAPE 05

Configuration et jonction du poste client

CLI-WIN-01 est renommé et configuré avec l'IP fixe 192.168.2.20/24. Point critique : le DNS préféré doit impérativement pointer vers 192.168.2.10 — c'est via DNS que Windows localise le domaine sisr.local.

La jonction s'effectue depuis Paramètres → Système → À propos de → Renommer ce PC (avancé) → Membre d'un domaine. On saisit sisr.local et les identifiants SISR\Administrateur. Un message de bienvenue confirme la réussite de la jonction.

Bilan 結果

◉ Difficulté rencontrée & résolue

Problème
Impossible de joindre le domaine depuis CLI-WIN-01 malgré ping et nslookup réussis.
Cause
DNS préféré de SRV-AD-01 configuré en 127.0.0.1 (loopback) : le serveur ne s'enregistrait pas correctement dans son propre DNS avec son IP réseau, rendant le DC introuvable pour les clients.
Solution
Modification du DNS préféré sur SRV-AD-01 de 127.0.0.1 vers 192.168.2.10. Jonction au domaine réussie immédiatement après correction.
Méthode
Analyse des messages d'erreur détaillés Windows, tests ping et nslookup depuis le client pour isoler la couche réseau de la couche DNS, puis vérification de la configuration DNS côté serveur.
